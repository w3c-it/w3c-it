HTML 5: principali novità #1

  • nuovi elementi per strutturare il contenuto
  • nuovi controlli e nuovi elementi di input per i form
  • accesso file/hardware : maggiore integrazione con il sistema operativo
  • estensione degli attributi di accessibilità a tutti gli elementi HTML5
  • grafica: nuovo elemento "canvas" supporta javascript per animazioni, ampio supporto alla gestione dello schermo, miglior supporto per SVG

HTML 5: principali novità #2

  • media: supporto nativo, semplificazione e potenziamento della gestione di audio e video
  • accesso ai sensori del computer / palmare / tablet / ...
  • local storage: memorizzazione persistente e di sessione sul browser (miglioramento vs cookie)
  • database locale: memorizzazione in database
  • Web Sockets: canali di comunicazione full-duplex bi-direzionale
  • Web workers: script in background e in parallelo

Archiviazione locale - Local storage

  • nuovo metodo per registrare informazioni
  • vantaggi rispetto ai cooki per la banda, velocità e la sicurezza:
    ogni cookie viene spedito avanti e indietro per ogni richiesta e per ogni file incluso (html,immmagine,css, javascript,...)
    una richiesta può richiedere molti secondi, è accessibile a pagine su qualsiasi server in un dominio/sottodominio.
  • In generale (a seconda del browser) il Local storage fornisce (fino a 5MB) di memorizzazione persistente dei dati (on in sessione del browser se specificato)
  • informazioni memorizzate utilizzando storage locale non accessibili da un altro host
<script>
localStorage.setItem('name', 'Hello world!'); // registra nel local storage la coppia chiave/valore
document.write(localStorage.getItem('name')); // Hello world
localStorage.removeItem('name'); // cancella la voce corrispondente dallo storage locale
</script>

IndexedDB

Per chi ha bisogno di informazioni strutturate e di più spazio

  • DB di tipo NoSQL
  • Javascript Native storage
  • DB di rercord mantengono valori semplici e oggetti gerarchici
  • Ciascun recordo consiste di una coppia chiave-valore
  • DB gestisce indici sui record che memorizza
  • Posizione dei record recuperata sia per chiave che per indice
  • Utilizzo asincrono: operazioni sono sempre transazioni dato che l'accesso può essere effettuato nello stesso momento da pagine differenti
  • Abilità di gestire oggetti blob (Binary Large Object)
  • Un DB creato da una pagina non può essere acceduto da un'altro host (per sicurezza)

Esempio IndexedDB #1

var request = indexedDB.open(dbName); // // create a database
request.onsuccess = function(event) {   // gestore della richiesta in caso di successo
  // gestione risultato
};
					
request.onerror = function(event) {   // gestore della richiesta in caso di errore
  alert("Errore DB: " + event.target.errorCode); 
};


request.onupgradeneeded = function(event) { //  necessario per modificare la struttura di un DB. Qui possiamo creare/cancellare objectStore e indici
  var myDB = event.target.result;

  // create the object stores   (una tabella in SQL)
  var personStore = myDB.createObjectStore("Person", {keyPath: "id"}); 

  // cre un indice per cercare persone per email, nessun duplicato ammissibile 
  personStore.createIndex("emailIndex", "email", { unique: true });

  // crea una transazione e registra un record
  var transaction = indexedDB.transaction("Person", "readwrite");
  var store = transaction.objectStore("Person");
  store.put({"id": "I2", "given_name" : "Anna", "family_name" : "Bianchi","email" : "anna.bianchi@w3c.it",... });

    # Cache settings for the manifest file
    <IfModule mod_expires.c>
    Header set cache-control: public
    ExpiresActive on

Web Socket

  • Nuovo protocollo di comunicazione full-duplex, può rimanere attivo permanentemente
  • Consente lo scambio di informazioni client / server
  • Inizialmente la connessione richiede uno scambio HTTP, dopo prosegue col nuovo protocollo ws o wss (ssl)
  • La richiesta parte del client: il client ottiene informazioni e poi chiude la connessione
  • Le connessioni con altri protocolli (es, SMTP / FTP / ...) non sono permesse da un WebSocket (sicurezza)
  • Video e foto veicolo sottomarino
