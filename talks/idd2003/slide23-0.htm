Non solo WCAG ...

  • Le "Web Content Accessibility Guidelines 1.0" Recommendation [WCAG10] sono un documento stabile
  • La tecnologia evolve, e i progettisti trovano maniere più efficaci per rendere accessibile il contenuto dei siti web
  • Le "Techniques" spiegano come soddisfare i requisiti definite nelle WCAG 1.0, ed evolvono al mutare della tecnologia:
    1. "Techniques for Web Content Accessibility Guidelines 1.0", è un documento propedeutico a tutti gli altri
    2. "Core Techniques for Web Content Accessibility Guidelines 1.0" ([WCAG10-CORE-TECHNIQUES]), esamina in generale il tema dell' accessibilità e le tecniche trasversali alle varie tecnologie (validazione, test, etc.)
    3. "HTML Techniques for Web Content Accessibility Guidelines 1.0" ([WCAG10-HTML-TECHNIQUES]), contiene esempi e strategie per creare contenuti Hypertext Markup Language (HTML) accessibili
    4. "CSS Techniques for Web Content Accessibility Guidelines 1.0" ([WCAG10-CSS-TECHNIQUES]), contiene esempi e strategie per aiutare gli autori a scrivere Cascading Style Sheets (CSS) come componente della progettazione di contenuti accessibili
    possono evolvere autonomamente
