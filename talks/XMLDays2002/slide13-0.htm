I punti deboli di HTML

  • HTML ottimizzato per facilità di apprendimento
    • Un solo tag set for tutte le applicazioni
    • Semantica predefinita per ogni tag
    • Strutture dati predefinite
    • Assenza di validazione formale
    • Supporto inadeguato per strutture a più livelli
  • HTML costituisce un compromesso tra potenza e semplicità d' uso
  • HTML molto adatto per applicazioni semplici
  • Poco adatto per applicazioni dai requisiti più complessi:
    • Raccolte di dati complesse o di ragguardevoli dimensioni
    • Dati con strutturazione ricca
    • Dati da utilizzare in vari modi
    • Dati con ciclo di vita lungo
