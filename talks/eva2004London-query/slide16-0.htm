Conclusion and future plans

  • A consistent framework for intelligent and controlled access to XML data collections has been implemented
  • RDF allowed the annotation of the (unchanged) schema
  • Administrator can point to any appropriate knowledge source (accessing thesauri via web services)
  • Administrator must just provide:
    • XML Schema
    • semantic description of XML elements and attributes
    This is just the Semantic Web Open World assumption
  • The interface is generalized:
    • any document collection (no effort required)
    • any search engine (implementation of the wrapper required)
  • Some features not yet implemented (but they are just "conventional development")
  • Future developments can consider:
    • moving towards a fully client oriented architecture:
      • XForms
      • local processing of thesauri (RDF description)
    • accessing heterogeneous document collections
    • inserting and updating documents
