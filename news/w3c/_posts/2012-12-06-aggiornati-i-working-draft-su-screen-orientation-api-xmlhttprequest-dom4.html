--- title: "Aggiornati i Working Draft su Screen Orientation API, XMLHttpRequest, DOM4" date: 2012-12-06 layout: page categories: ---

Il Web Applications Working Group ha pubblicato oggi tre Working Draft:

  • Screen Orientation API. La Screen Orientation API ha lo scopo di fornire un'interfaccia per far sì che le applicazioni Web possano leggere lo stato dell'orientamento dello schermo, siano informate quando questo stato cambia e possano bloccare l'orientamento dello schermo in uno stato specifico.
  • XMLHttpRequest. La specifica XMLHttpRequest definisce una API che fornisce al client funzionalità a livello di script per trasferire dati tra un client e un server.
  • DOM4, pubblicato in cooperazione con il Web Hypertext Application Technology Working Group. Il DOM definisce il document model e gli eventi che la piattaforma Web utilizza. Il DOM è un'interfaccia indipendente dal linguaggio e dalla piattaforma che permette a programmi e script di accedere al contenuto e alla struttura dei documenti e di aggiornarli dinamicamente.
