--- title: "Call per revisione del Gruppo di Lavoro Web Performance su User Timing and Performance Timeline, call per implementazione di Timing control for script-based animations" date: 2013-10-31 categories: ---

Il  Gruppo di Lavoro Web Performance ha pubblicato oggi due proposte di Raccomandazione e invita all'implementazione della Raccomandazione Candidata:

  • Una Proposta di Raccommandazione su User Timing, che definisce un'interfaccia in grado di aiutare gli sviluppatori Web nel misurare la performance delle loro applicazioni dando loro accesso a un timestamp di alta precisione. I commenti sono benvenuti fino al 28 Novembre.
  • Una proposta di Raccommandazione su Performance Timeline. Questa specifica definisce un'interfaccia unificata per immagazzinare e ricercare dati metrici sulla performance.  Questa specifica non copre singole interfacce metriche di performance. I commenti sono benvenuti fino al 28 novembre.
  • Una raccomandazione Candidata su Timing control for script-based animations. Questo documento definisce un API che gli autori di pagine Web possono utilizzare per creare animazioni basate su script in cui l'interprete è in grado di limitare la frequenza di aggiornamento dell'animazione. Lo user agent è nella posizione migliore per determinare il tasso ideale di animazione a seconda che la pagina sia attualmente in secondo piano o in primo piano, a seconda del carico sulla CPU è, e così via. L'utilizzo di questa API dovrebbe quindi tradursi in un utilizzo più appropriato della CPU da parte del browser.
