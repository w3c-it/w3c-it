--- title: "Il W3C Invita alle Implementazioni del Widget Access Request Policy" date: 2010-04-20 layout: page categories: ---

Il Web Applications Working Group invita all'implementazione della Candidate Recommendation di Widget Access Request Policy. Gli user agent che eseguono i  widget devono garantire l'accesso alle API potenzialmente sensibili (rubrica, agenda, file system, etc.) che mostrano dati che non dovrebbero essere mostrati senza il consenso dell'utente. Lo scopo di questa specifica è definire il modello di sicurezza per l'interazione delle reti all'interno di un widget che ha accesso alle informazioni sensibili. Esso permette a un widget di dichiarare il suo intento di accedere a specifiche risorse di rete cosicché un sistema possa controllarlo. Si seguano gli sviluppi dell' implementation report del gruppo e si consulti Rich Web Client Activity per avere più informazioni.
