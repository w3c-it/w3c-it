--- title: "Core Accessibility API Mappings 1.1 (Core-AAM) First Public Working Draft and WAI-ARIA 1.1 updated Working Draft" date: 2014-06-12 layout: page categories: w3c ---

Il Protocols and Formats Working Group ha pubblicato oggi la prima Public Working Draft sulle Core Accessibility API Mappings 1.1 (Core-AAM), a supporto della versione aggiornata della Working Draft sulle Accessible Rich Internet Applications (WAI-ARIA) 1.1. WAI-ARIA fornisce un'ontologia di ruoli, stati e proprietà che definiscono gli elementi accessibili delle user interface. WAI-ARIA è progettato per migliorare l'accessibilità e l'interoperabilità del web content, in particolare delle web applications. Core-AAM descrive come gli user agents debbano esporre le semantiche dei content languages alle accessibility APIs attraverso tecnologie cross multiple content (incluse quelle di WAI-ARIA). Core-AAM serve da base per altre specifiche che estendono specifiche tecnologie. Maggiori informazioni nella call for review e-mail e su Web Accessibility Initiative (WAI).
