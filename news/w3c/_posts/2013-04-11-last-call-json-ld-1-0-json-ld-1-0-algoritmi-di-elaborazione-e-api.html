Last Call: JSON-LD 1.0, JSON-LD 1.0 Algoritmi di Elaborazione e API

11 Aprile 2013

Oggi il RDF Working Group e il JSON-LD Community Group hanno pubblicato due Last Call Working Drafts:

  • JSON-LD 1.0. JSON-LD armonizza la rappresentazione dei Linked Data in JSON, descrivendo un formato comune di rappresentazione JSON per esprimere grafi diretti; combinando Linked Data e non-Linked Data in un singolo documento. La sintassi è progettata per non interferire con sistemi già in dispiegati su JSON e fornisce un facile percorso di aggiornamento da JSON a JSON-LD. È rivolta principalmente per dare un modalità di utilizzo dei Linked Data in ambiente di programmazione Web-based, per costruire servizi Web di Linked Data interoperabili e per memorizzare Linked Data in motori di archiviazione JSON-based. JSON-LD è capace di serializzare ogni tipo di grapo o dataset RDF e molti ma non tutti i documenti JSON-LD, , possono essere trasformati direttamente in RDF.
  • JSON-LD 1.0: Algoritmi e API. La specifica degli Algoritmi delle API JSON-LD 1.0 descrive Algoritmi utili a lavorare con i dati JSON-LD. inoltre definisce una Application Programming Interface che può essere usata per trasformare i documenti JSON-LD e facilitare il lavoro in ambienti di programmazione come JavaScript, Python e Ruby.

Questa è la pubblicazione di una Last Call per entrambe le specifiche. tutto il sostanziale lavoro tecnico è completo. È incoraggiato il feedback per entrambe le specifiche, che va inviato e public-rdf-comments@w3.org. Il periodo di Last Call finirà tra 4 settimane, il 10 Maggio 2013
