XML Signature e XML Encryption sono W3C Recommendations

16 Aprile 2013

L'XML Security Working Group ha oggi pubblicato tre W3C Recommendation:

  • XML Signature Syntax e Processing Version 1.1. Questo documento specifica le regole di elaborazione e la sintassi della firma digitale XML. Le firme XML forniscono integrità, autenticazione del messaggio e/o servizi di autenticazione del firmatario per dati di qualsiasi tipo, sia situati all'interno dell'XML che include la firma, sia situati altrove.
  • XML Encryption Syntax e Processing Version 1.1. Questo documento specifica un processo per l'encrypting dei dati e per reppresentare il risultato in XML. I dati possono essere in una varietà di formati, inclusi octet streams e altri dati non strutturati, oppure formati di dati strutturati come documenti XML, un elemento XML o il contenuto di un elemento XML. Il resultato dell'encrypting dei dati è un elemento XML Encryption che contiene o riferisce i dati cifrati.
  • XML Signature Properties. Questo documento delinea la sintassi, le regole di elaborazione e un namespace associato per le proprietà da usare nelle XML Signature. Queste possono essere composte con qualsiasi versione di XML Signature, utilizzando l'elemento XML SignatureProperties. Queste proprietà sono destinate a soddisfare i requisiti di firma del codice.

Il gruppo ha oggi anche pubblicato tredici Working Group Notes. Per saperne di più vai al link Security Activity.
