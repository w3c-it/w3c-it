--- title: "XQuery, XSLT 2 e XPath 2 sono W3C Recommendation" date: 2007-01-22 layout: page categories: w3c --- Il World Wide Web Consortium ha pubblicato otto nuovi standard nella famiglia XML per data mining, trasformazione di documenti, e computing aziendale, da Web service a database. "Oltre 1.000 commenti da parte degli sviluppatori ci hanno aiutato a garantire un set elastico e implementabile di tecnologie database," ha dichiarato Jim Melton (Oracle). XSLT trasforma i documenti in differenti markup o formati. XML Query può effettuare ricerche, query e collegamenti attraverso collezioni di documenti. Usando espressioni XPath , XSLT 2 ed XQuery si può operare su documenti XML, database XML, database relazionali, motori di ricerca e object repository. Maggiori informazioni sulla press release e sulla pagina dei testimonial e l'XML home page. (Archivio News inglese, italiano)
  • XQuery 1.0: An XML Query Language:
    Una sintassi XML-aware per interrogare collezioni di dati strutturati e semi-strutturati, sia localmente che nel Web
  • XSL Transformations (XSLT) Version 2.0:
    Trasforma le istanze data model (XML and non-XML) in altri documenti, inclusi XSL-FO per la stampa
  • XML Path Language (XPath) 2.0:
    Espressione di sintassi per indirizzare alle parti dei documenti XML
  • XQuery 1.0 and XPath 2.0 Functions and Operators:
    Funzioni che si possono richiamare in espressioni XPath ed operazioni che si possono effettuare su XPath 2.0 data type
  • XQuery 1.0 and XPath 2.0 Data Model (XDM):
    Rappresentazioni per entrambi i codici XML and non-XML
  • XSLT 2.0 and XQuery 1.0 Serialization:
    Come raccogliere valori XSLT 2.0 e XML Query in XML, HTML o come testo
  • XML Syntax for XQuery 1.0 (XQueryX):
    Una rappresentazione precisa XML dell'XML Query language, adatta per l'elaborazione della macchina e l'introspezione
  • XQuery 1.0 and XPath 2.0 Formal Semantics:
    Il type system usato in XQuery e XSLT 2 via XPath definito precisamente per gli implementatori
