--- title: "Call per Revisione: XML Signature 1.1, XML Encryption 1.1, pubblicate le raccomandazioni proposte per l'elemento XML Signature Properties " date: 2013-01-24 layout: page categories: ---

L' XML Security Working Group oggi ha pubblicato tre Proposte di Recommendation:

  • XML Signature Syntax and Processing Version 1.1. Questo documento specifica le regole procedurali e la sintassi della firma digitale XML. Le Firme XML garantiscono integrità, autenticazione dei messaggi e/o servizio di autenticazione del firmatario per qualsiasi tipo di dati, sia che si trovino nell'XML che include la firma sia altrove. I commenti sono benvenuti entro il 25 febbraio.
  • XML Encryption Syntax and Processing Version 1.1. Questo documento specifica un processo per criptare dati e rappresentare il risultato in XML. I dati possono essere in vari formati, incluse stringhe di ottetti e altri dati non strutturati, o formati di dati strutturati come documenti XML, un elemento XML, o il contenuto di un elemento XML. Il risultato dei dati crittografati è un elemento XML Encryption che contiene o fa riferimento ai dati cifrati. I commenti sono benvenuti entro il 25 febbraio.
  • XML Signature Properties. Questo documento delinea uno standard proposto per la sintassi e le regole procedurali dell'XML Signature Properties e un namespace associato per queste proprietà. L'intento è che questi possono essere creati con qualsiasi versione di XML Signature usando l'elemento XML SignatureProperties. Queste proprietà sono destinate a soddisfare i requisiti di firma  del codice. I commenti sono benaccetti entro il 25 febbraio.

Il gruppo ha anche pubblicato sei Note del Gruppo di Lavoro:

  • Functional Explanation of Changes in XML Encryption 1.1 fornisce un riassunto dei cambiamenti non-editoriali nell'XML Encryption 1.1 dal XML Encryption Recommendation.
  • XML Security 1.1 Requirements and Design Considerations riassume gli scenari, le decisioni di design e i requisiti per l'XML Signature e le specifiche del XML canonico, per guidare i lavori in corso del W3C nella revisione di queste specifiche.
  • XML Security Algorithm Cross-Reference riassume gli identificatori URI dell'algoritmo XML Security e le specifiche associate a questi.
  • XML Security Generic Hybrid Ciphers specifica una sintassi XML e le regole di procedura per codici ibridi generici e meccanismi di incapsulamento a chiave e inoltre riserva identificatori per gli algoritmi.
  • XML Security RELAX NG Schemas fornisce schemi non-normativi RELAX NG nella sintassi compatta così come nella sintassi XML.
  • XML Signature Best Practices raccoglie le best practices per sviluppatori e utenti della specifica XML Signature, alcune delle quali volte a migliorare la sicurezza e ridurre gli attacchi.
