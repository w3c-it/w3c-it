--- title: "Il Web Applications Working Group ha aggiornato Streams API, Quota Management API, DOM Level 3 Events Specification e UI Events" date: 2013-11-06 layout: page categories: w3c ---

Il Web Applications Working Group ha pubblicato quattro Working Drafts:

  • Streams API. Questa specifica fornisce una API per rappresentare nelle applicazioni Web i dati in forma binaria e di stringhe come oggetti Stream, e di costruire e leggere il loro contenuto in modo programmatico. Include interfacce Stream, StreamConsumeResult e StreamReadType, ed estensioni ad  XMLHttpRequest e a URL.createObjectURL e URL.revokeObjectURL. Questa API è progettata per essere usata insieme ad altre API ed elementi delle piattaforme Web, come: File, XMLHttpRequest, postMessage e Web Workers.
  • Quota Management API. Questa specifica definisce una API per gestire l'uso e la disponibilità di risorse locali di storage, e definisce un mezzo tramite il quale uno user agent (UA) può dare alle applicazioni Web il permesso di usare una maggior quantità di spazio locale, in modo permanente o temporaneo.
  • Document Object Model (DOM) Level 3 Events Specification. Questa specifica definisce il Document Object Model Events Level 3, un sistema ad eventi indipendente dalla piattaforma e dal linguaggio che permette la registrazione di gestori di eventi, descrive il flusso di eventi attraverso una struttura ad albero, e fornisce un'informazione contestuale di base per ogni evento. Il Document Object Model Events Level 3 si basa su Document Object Model Events Level 2.
  • UI Events. Questa specifica estende gli eventi e le caratteristiche definite in DOM Events Level 3.
