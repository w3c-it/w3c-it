--- title: "Il W3C invita alle implementazioni di Cross-Origin Resource Sharing " date: 2013-01-29 categories: ---

Il Web Applications Working Group e il Web Application Security Working Group hanno pubblicato una Candidate Recommendation sul Cross-Origin Resource Sharing. Questo documento definisce un meccanismo per abilitare le richieste client-side cross-origin. Specifiche che abilitano un API a fare richieste cross-origin alle risorse possono usare l'algoritmo definito da questa specifica. Se questo tipo di API è usato su risorse http://example.org, una risorsa su http://hello-world.example può optare se usare il meccanismo descritto da questa specifica (per esempio, specificando Access-Control-Allow-Origin: http://example.org nell'header della risposta), cosa che consentirebbe a questa risorsa di essere prelevata cross-origin da http://example.org. Maggiorni informazioni su  Rich Web Client Activity e  Security Activity.
