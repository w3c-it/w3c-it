Pubblicato il draft di WebDriver

17 gennaio 2013

Il Gruppo di lavoro Browser Testing and Tools ha reso publico un working draft per WebDriver. Questa specifica definisce le API di WebDriver, una interfaccia indipendente da piattaforme e linguaggi che consente a programmi o script di ispezionare all'interno o controllare il comportamento di un browser web. Le API WebDriver sono concepite principalmente per permettere agli sviluppatori di scrivere test che automatizzino un browser tramite processi di controllo separati, ma possono anche essere implementate in modo tale da consentire a script in-browser di controllare un browser eventualmente separato. Le API WebDriver sono definite da un insieme di interfacce per scoprire e manipolare elementi DOM in una pagina, e per controllare il comportamento del browser che li contiene. La specifica include anche una serializzazione di riferimento (verso JSON) delle invocazioni e risposte dell'interfaccia che potrebbero risultare utili per i produttori di browser. Per saperne di più vai a Web Testing Activity.
