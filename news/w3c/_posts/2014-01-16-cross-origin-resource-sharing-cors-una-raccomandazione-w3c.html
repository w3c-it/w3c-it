Cross-Origin Resource Sharing (CORS) is a W3C Recommendation

16 January 2014

Il Web Applications (WebApps) Working Group e il Web Application Security (WebAppSec) Working Group hanno pubblicato oggi una Raccomandazione su Cross-Origin Resource Sharing (CORS).Questo documento definisce un meccanismo per abilitare le richieste cross-origin  lato-client. Le specifiche che abilitano un API a fare richieste cross-origin alle risorse possono usare gli algoritmi definiti da questa specifica. Se un tale API è usato su risorse http://example.org, una risorsa su http://hello-world.example può optare di usare il meccanismo descritto da questa specifica ( ad es. specificando Access-Control-Allow-Origin: http://example.org as response header), che permetterebbe a quella risorsa di essere presa cross-origin da http://example.org. Per saperne di più circa la Rich Web Client Activity e la Security Activity.
